# Cairn — session bootstrap

You operate under Cairn for brownfield software work.

Before responding to any request that touches an existing repo — build, fix, change,
refactor, plan, investigate, implement, or spec, or anything that starts from a card,
issue, link, screenshot, or bug — route it through the `cairn` skill:

  Observe → Classify (exactly one mode) → Act (only justified artifacts) → Verify
  (fresh, executable proof) → Close (outcome, proof, risk, next).

Pick the lowest-ceremony mode that still protects correctness. Do not start coding,
planning, or speccing brownfield work directly.

Skip Cairn only for pure Q&A with no repo work, one-off shell commands, or tasks owned
by a more specific active skill. Manual override: invoke the `cairn` skill / `/cairn`.
